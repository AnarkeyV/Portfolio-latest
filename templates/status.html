How This Is Monitored

This page is powered by a local Python health agent. The agent checks the app, health endpoint, Docker Compose state, and public route availability, then writes a local JSON status file for Flask to render.

Step 01

Run safe health checks against the local Flask service and operational endpoints.

Step 02

Summarize the result into a public-safe status report without exposing raw logs.

Step 03

Render the latest status through Flask so visitors can see the portfolio’s operational state.

Note: Internal IP addresses, Windows usernames, local folder paths, container IDs, tunnel names, and raw logs are intentionally not shown here. The Ollama/local LLM layer is not required for the website to stay online. The public dashboard is based on deterministic health checks, not AI guesses.
